```typescript
export class InputManager {
  private readonly heldKeys: Set<string> = new Set();
  private readonly justPressedKeys: Set<string> = new Set();

  constructor() {
    window.addEventListener('keydown', (e) => {
      if (!this.heldKeys.has(e.code)) this.justPressedKeys.add(e.code);
      this.heldKeys.add(e.code);
      if (e.code === 'Space') e.preventDefault();
    });
    window.addEventListener('keyup', (e) => { this.heldKeys.delete(e.code); });
  }

  public isDown(code: string): boolean { return this.heldKeys.has(code); }
  public justPressed(code: string): boolean { return this.justPressedKeys.has(code); }
  public anyKeyJustPressed(): boolean { return this.justPressedKeys.size > 0; }
  public clearJustPressed(): void { this.justPressedKeys.clear(); }
}
```

Extend InputManager with full gamepad support. The key insight: gamepad state is synthesized into existing heldKeys/justPressedKeys Sets so all game states work automatically.

**Game.ts wiring (MUST be done first to understand constructor approach):**
Currently Game.ts creates InputManager in the constructor (line 41) but hudRoot is only available in init() (line 62). Two clean approaches:
1. Move `this.input = new InputManager()` into init() after hudRoot is created, changing the field from readonly to late-init — but this changes a public API.
2. Better: keep InputManager constructor accepting an optional toast. Create the GamepadToast in Game.init() and pass it to InputManager via a new `setToast(toast: GamepadToast)` method.
3. Simplest: InputManager constructor takes an optional `hudRoot?: HTMLElement` and creates its own GamepadToast internally if provided. Game.ts changes to `this.input = new InputManager()` in constructor (unchanged), then in init() calls `this.input.initGamepad(hudRoot)`.

**Use approach 3:** Add a public `initGamepad(hudRoot: HTMLElement): void` method to InputManager that creates the GamepadToast and sets up gamepad event listeners. This keeps the constructor signature backward-compatible (no breaking change) and defers DOM access to when hudRoot is available. In Game.ts init(), after `const hudRoot = document.getElementById('hud') as HTMLElement;` (line 62), add `this.input.initGamepad(hudRoot);`.

**InputManager changes:**

**New public property:**
- `activeInputDevice: 'keyboard' | 'gamepad' = 'keyboard'` — tracks last-used input device

**Existing constructor modification:**
- Add `this.activeInputDevice = 'keyboard';` in the keydown handler (one line added)

**New public method `initGamepad(hudRoot: HTMLElement): void`:**
This sets up all gamepad infrastructure:
- Create `this.toast = new GamepadToast(hudRoot)`
- Add `gamepadconnected` window listener:
  - Store `this.activeGamepadIndex = e.gamepad.index`
  - Initialize `this.prevGamepadButtons = e.gamepad.buttons.map(b => b.pressed)` (prevents double-input on first frame — Pitfall 3 from research)
  - Set `this.activeInputDevice = 'gamepad'`
  - Extract readable name via regex: `e.gamepad.id.replace(/^[\da-f]+-[\da-f]+-/i, '').slice(0, 40)` (Pitfall 6)
  - Show connect toast: `this.toast.show('CONTROLLER CONNECTED \u2014 ${name}', 'connect')`
  - Check compatibility: if `e.gamepad.mapping !== 'standard'`, show warning toast after a short delay (setTimeout 100ms) so it appears after the connect toast: `this.toast.show('WARNING: non-standard controller \u2014 inputs may be incorrect', 'warning')`
  - Play menuNav SFX: `audioManager.playSfx('menuNav')`
- Add `gamepaddisconnected` window listener:
  - If `this.activeGamepadIndex === e.gamepad.index`: set `activeGamepadIndex = null`, clear `prevGamepadButtons = []`, call `this._clearSynthesizedKeys()`, set `activeInputDevice = 'keyboard'`
  - Show disconnect toast: `this.toast.show('CONTROLLER DISCONNECTED', 'disconnect')`
  - Play menuNav SFX: `audioManager.playSfx('menuNav')`

**New private fields:**
- `toast: GamepadToast | null = null`
- `activeGamepadIndex: number | null = null`
- `prevGamepadButtons: boolean[] = []`
- `prevStickLeft = false`, `prevStickRight = false`, `prevStickUp = false`, `prevStickDown = false` — for stick justPressed edge detection
- `static readonly DEADZONE = 0.20`
- `static readonly GAMEPAD_CODES = ['ArrowLeft', 'ArrowRight', 'ArrowUp', 'ArrowDown', 'Space', 'Escape', 'KeyP', 'KeyC']` — all codes that can be synthesized from gamepad

**clearJustPressed() modification:**
- After `this.justPressedKeys.clear()`, call `this._pollGamepad()` — poll once per fixed step, on the same clock as the game loop

**New private method `_pollGamepad()`:**
- If `this.activeGamepadIndex === null`, return early
- Call `navigator.getGamepads()` fresh each frame (never cache — research anti-pattern)
- Get `gp = gamepads[this.activeGamepadIndex]`, return if null
- Track if any gamepad input is active (any button pressed or any stick axis beyond deadzone); if so, set `this.activeInputDevice = 'gamepad'`
- Button polling loop: iterate `gp.buttons`, for each button with a mapped code (via `_buttonToCode`):
  - If `isPressed && !wasPressed` -> add code to `justPressedKeys` (edge detection)
  - If `isPressed` -> add code to `heldKeys`
  - If `!isPressed` -> delete code from `heldKeys`
  - Update `prevGamepadButtons[i] = isPressed`
- Call `this._synthesizeStick(gp.axes[0], gp.axes[1])` for left stick

**New private static method `_buttonToCode(index: number): string | null`:**
Per user decision button mapping:
- 0 (A/Cross) -> 'Space' (fire + confirm)
- 1 (B/Circle) -> 'Escape' (back/cancel)
- 3 (Y/Triangle) -> 'KeyC' (continue on GameOver)
- 5 (RB/R1) -> 'Space' (alternate fire)
- 9 (Start/Options) -> 'KeyP' (pause)
- 12 (D-pad Up) -> 'ArrowUp'
- 13 (D-pad Down) -> 'ArrowDown'
- 14 (D-pad Left) -> 'ArrowLeft'
- 15 (D-pad Right) -> 'ArrowRight'
- All others -> null (unmapped)

**New private method `_synthesizeStick(x: number, y: number)`:**
Per user decision: binary threshold, always read horizontal component even at diagonals.
- Calculate `magnitude = Math.sqrt(x*x + y*y)`
- If magnitude < DEADZONE: delete ArrowLeft, ArrowRight, ArrowUp, ArrowDown from heldKeys; reset all prevStick* to false; return (no drift)
- Horizontal (always read): if x < 0 -> add ArrowLeft, delete ArrowRight; else -> add ArrowRight, delete ArrowLeft
- Vertical (for menu nav): if y < -DEADZONE -> add ArrowUp, delete ArrowDown; if y > DEADZONE -> add ArrowDown, delete ArrowUp; else delete both vertical
- justPressed edge detection for stick: compare current direction state to prevStick* booleans. If direction is newly active, add to justPressedKeys. Update prevStick* for next frame. This is critical — TitleState menu navigation reads justPressed('ArrowDown') not isDown('ArrowDown').

**New private method `_clearSynthesizedKeys()`:**
- Iterate GAMEPAD_CODES and delete each from heldKeys (prevents phantom drift on disconnect — Pitfall 2)
- Reset all prevStick* booleans to false

**Import:** Add `import { audioManager } from '../systems/AudioManager';` and `import { GamepadToast } from '../ui/GamepadToast';` at top. Per research open question #1, AudioManager does not import InputManager, so no circular dependency.

**IMPORTANT:** The existing keyboard listeners must remain unchanged. Gamepad synthesis ADDS to the same Sets that keyboard events write to. When gamepad disconnects, only gamepad-synthesized codes are cleared — keyboard-held keys remain intact.

**Game.ts changes (minimal):**
- Add import: `import { GamepadToast } from '../ui/GamepadToast';` (not actually needed — InputManager handles it internally)
- In `init()`, after line 62 (`const hudRoot = ...`), add: `this.input.initGamepad(hudRoot);`
