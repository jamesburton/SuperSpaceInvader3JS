In `src/core/SceneManager.ts`:
- Add a `reinitCrt(tier: number | null, intensity: number): void` method that:
  1. Disposes existing CRT manager if present: `this.crtManager?.dispose(); this.crtManager = null;`
  2. Calls `this.initCrt(tier, intensity)` to create a new one with the new tier.
  This is needed because when a player purchases a CRT tier in the meta shop, the CRT pass needs to be rebuilt with the new tier parameters (since tier affects which effects are created). The EffectComposer supports removePass()/addPass() mid-session.
- Actually, since `initCrt` creates a new CRTManager and adds a new pass, we need to handle the case where an old pass exists. Modify `initCrt` to first dispose the old CRT:
  ```typescript
  public initCrt(tier: number | null, intensity: number): CRTManager | null {
    // Dispose existing CRT if reinitializing
    if (this.crtManager) {
      this.crtManager.dispose();
      this.crtManager = null;
    }
    if (!this.bloomEffect || tier === null || tier < 1) return null;
    this.crtManager = new CRTManager();
    this.crtManager.init(this.bloomEffect.composer, this.camera, tier, intensity);
    return this.crtManager;
  }
  ```
  Wait — `dispose()` on the EffectPass does NOT remove it from the EffectComposer. We need `composer.removePass()`. Update CRTManager to accept and store a reference to the composer, and have a `remove(composer)` or update the dispose pattern.

  Better approach: Add a `removeFromComposer(composer: EffectComposer)` step. Actually, simplest approach: CRTManager stores the composer reference internally, and dispose() removes the pass from the composer AND disposes it:
  ```typescript
  // In CRTManager:
  private composer: EffectComposer | null = null;

  public init(composer: EffectComposer, ...): void {
    this.composer = composer;
    // ... create effects and pass ...
    composer.addPass(this.pass);
  }

  public dispose(): void {
    if (this.pass && this.composer) {
      this.composer.removePass(this.pass);
    }
    this.pass?.dispose();
    this.scanline = null;
    this.chromatic = null;
    this.pass = null;
    this.composer = null;
  }
  ```
  Update the CRTManager.ts file (from Plan 08-03) to store the composer reference. This is a modification to an 08-03 file, but since 08-04 depends on 08-03, this is fine.

  Then SceneManager.initCrt() handles reinit cleanly by calling dispose() first (which removes old pass) then creating new.
